# Writeup 模板使用说明

## 这个模板怎么用

一句话：**做题过程中同步往里填，比赛结束前自然就写完了。**

---

## 什么时候该填什么

### 做题前（开一道新题时）

1. 在 `README.md` 的"解题概述"和"关键改进"里新增一道题的小节
2. 在 `src/` 下建这道题的代码子目录（如 `src/ns_2026_xx/`）
3. 复制一份 `NS-2026-XX_submission/` 改成实际题号

### 做题中（每次有推进时）

| 发生了什么事 | 要去填什么 |
|-------------|-----------|
| 有了一个初步方案 | 写"解题概述"里的任务理解和核心思路 |
| 做了一次有效改进（分数提升） | 写"关键改进"一条：改了什么 + 分数变化 |
| 使用 AI 辅助 | 同步记录到"AI 使用声明"里 |
| 在平台上提交了一次 | 截图保存到 `screenshots/`（01 最终提交记录） |
| 跑训练/推理 | 终端输出截图保存到 `screenshots/`（03 训练过程） |
| 算力环境确定下来 | 记下 OS、Python 版本、GPU 型号到"验证与复现" |

### 提交最终文件时

1. 算 SHA256，截图保存到 `screenshots/`（04 文件哈希）
2. 把提交文件放到对应题目的 `_submission/` 目录

### 比赛结束后

1. 运行环境截图（02）：跑一下 `nvidia-smi`、`lscpu` 截图
2. 检查所有部分是否填完
3. 确认 `src/` 下的代码没有硬编码路径
4. 确认 `requirements.txt` 包含了所有依赖
5. 打包成 `[队伍ID]_Writeup.zip` 提交

## 模板结构速查

```
_Writeup_Template/
├── README.md                # ← 主文档，覆盖所有题目（6 部分）
├── src/                     # ← 可复现代码
│   ├── README.md            # 复现步骤
│   └── requirements.txt     # 依赖清单
├── screenshots/             # ← 截图统一放这里
│   └── README.md            # 截图清单和命名规则
└── NS-2026-XX_submission/   # ← 每道题一份，放最终提交文件
    └── README.md
```

## 常见疑问

### 这个模板要等到比赛结束后再填吗？

不要。等你做完题再回想改了什么、分数变化多少、用了哪些 AI，大概率记不全。**每做完一次有效操作就顺手填一笔**，最后只需要检查有没有遗漏。

### 一道题由两个人做，谁负责填模板？

建议明确分工。参考之前的建议：**主力专注做题，辅助同步填写模板**。这样主力不用分心记录，辅助有事可做，最后也不会缺材料。

### 只做了部分题怎么办？

在 `README.md` 里只写你做过的题，没做的题删掉对应小节即可。

### 模板里的题目示例（NS-2026-XX）怎么改？

把 `XX` 替换成实际题号，比如第一题是 `NS-2026-01`。题号以平台为准。

### 截图什么时候截最好？

| 截图 | 最佳时机 |
|------|----------|
| 01 最终提交记录 | 每次提交后立刻截，选最高分那次即可 |
| 02 运行环境 | 开赛时截一次，比赛结束后再截一次确认没变 |
| 03 训练/推理过程 | 跑实验时顺手截 |
| 04 SHA256 | 算出 hash 后顺手截 |

### 代码要写到什么程度？

要能让审核的人**拿公开数据在你的说明下一步步跑出合法提交**。具体要求：
- `pip install -r requirements.txt` 能装好环境
- `python train.py` 能训练
- `python infer.py` 能生成提交文件
- 没有 `C:/Users/你的名字/...` 这种绝对路径

## 最后一步：打包

```bash
# 把 _Writeup_Template 重命名为 [队伍ID]_Writeup
# 确认内容无误后压缩
zip -r [队伍ID]_Writeup.zip [队伍ID]_Writeup/
```

提交后等待审核，审核期间保持联系畅通即可。
